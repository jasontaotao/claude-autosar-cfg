## Background

v1.14.0 MINOR ship deferred 3 items to v1.14.1 PATCH per
[[claude-autosarcfg-v1-14-0-shipped]] § Deferred to v1.14.1 backlog:

- **S2+** dangling `#include` — `EcuC_Cfg.h` emits `&Os_OsCore_OsCore_0`
  but does not `#include "Os/Os_Cfg.h"`; downstream compile fails
  with undeclared symbol.
- **SEC3** still defensive-only — `validateHeaderPath` is exported in
  `handlebars-helpers.ts:43` but never called on parser output.
- **Mcu recursion parity** — `modules/mcu.ts:169-179` still uses flat
  for-loop; only `EcuC` is wired to `walkContainers` from v1.14.0 S8.
  Mcu also does not consume `references[]` (no S2 parity).

Root cause for the first two: BSWMD parser (`src/core/project/bswmd.ts`)
does not surface `moduleHeader` / `includes[]` from
`<ECUC-MODULE-DEF>/<HEADER>` and `<STD-INCLUDES>/<STD-INCLUDE>`
elements. v1.14.1 PATCH-G closes the trio by extending parser +
generator + validator in one batch.

## Goal

Generated `EcuC_Cfg.h` / `Mcu_Cfg.h` are self-contained and compile
without manual `#include` patching. BSWMD parser carries module
metadata end-to-end. SEC3 validation is wired.

## Architecture

End-to-end data flow (no new abstractions — all wiring uses
existing pipeline stages):

```
Stage 1: Load
  ARXML (ecuc-bswmd.arxml)
    → bswmd.ts parseModuleDef() parses <HEADER> + <STD-INCLUDES>
    → BswmdModuleDef { shortName, moduleHeader?, includes?, ... }
  normalize.ts
    → bswmdIndex: Map<shortName, BswmdModuleDef>
    → ecucValues: Map<shortName, EcucModuleConfigurationValues>
       (includes references[] from S2)

Stage 2: Validate (G4 wire-up)
  pipeline.ts Stage 1 validator chain
    → validateModuleHeaderPaths(bswmdIndex)   ← NEW (G4)
       - For every moduleDef: validateHeaderPath(moduleHeader)
       - For every moduleDef.includes[]: validateHeaderPath(include)
       - Failure → push ERROR { code: 'BSW-SEC-002' }
    → existing validators (S9 instance, S10 cross-type, etc.)
  S6 early-break: if any ERROR → skip Stage 2

Stage 3: Generate (G2 + G3)
  EcuCGenerator.emit (modules/ecuc.ts)
    1. Build includes[]:
       a. existing fixture-derived includes (empty today)
       b. NEW (G2): walk eVals.references[]
          for each ref.targetModule:
            hdr = bswmdIndex[ref.targetModule]?.moduleHeader
            if hdr && !includesSet.has(hdr) && validateHeaderPath(hdr):
              includes.push(hdr); includesSet.add(hdr)
       c. if ref.targetModule loaded but moduleHeader undefined:
          push ERROR { code: 'BSW-SEC-004' }
    2. Build referenceDecls[] (S2 unchanged)
    3. walkContainers(eDef.containers, ...) (S8 unchanged, already wired)
    4. Render cfg.h.hbs with { includes, referenceDecls, ... }
       (template already has {{#each includes}} — 0 template change)

  McuGenerator.emit (modules/mcu.ts)  ← parity with EcuC
    1-3. Same shape as EcuC
       (G3: switch flat for-loop → walkContainers;
        G3: consume mVals.references[])
    4. Render cfg.h.hbs
```

## Error Handling

| Trigger                                                          | Severity  | Code                                    | Behavior                                                |
| ---------------------------------------------------------------- | --------- | --------------------------------------- | ------------------------------------------------------- |
| `moduleHeader` contains `..` / starts with `/` / fails whitelist | **ERROR** | `BSW-SEC-002`                           | S6 early-break Stage 2                                  |
| `moduleHeader` non-string or empty                               | **ERROR** | `BSW-SEC-002`                           | same                                                    |
| `<STD-INCLUDE>` element has invalid char                         | **ERROR** | `BSW-SEC-002`                           | same                                                    |
| `<STD-INCLUDE>` has empty `<SHORT-NAME/>`                        | **WARN**  | `BSW-SEC-003`                           | pipeline continues; strict mode upgrades to ERROR       |
| `ref.targetModule` loaded but `moduleHeader` undefined           | **ERROR** | `BSW-SEC-004`                           | S6 early-break; user told which module needs `<HEADER>` |
| `ref.targetModule` not in `bswmdIndex`                           | **ERROR** | `ECUC-GEN-REF-UNRESOLVED` (existing S9) | unchanged                                               |
| Same `ref.targetModule` referenced multiple times                | no-op     | —                                       | `Set` dedupes the `#include`                            |

`strict: true` (CLI flag) promotes `BSW-SEC-003` from WARN → ERROR.
`BSW-SEC-002` and `BSW-SEC-004` are always ERROR.
