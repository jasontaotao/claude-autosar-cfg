## Architecture

### R-1 — Delete `walkContainers`

**Current state** (verified by verifier):

```bash
$ grep -rn "walkContainers\b" src/ | grep -v "walkContainersWith"
src/core/generator/emit/container.ts:70  # definition
src/core/generator/emit/container.ts:77  # internal recursion
src/core/generator/__tests__/emit-container-recursion.test.ts:12,17,20,52,66,81,89  # tests only
```

After v1.14.2 H3 (EcuC swap to `walkContainersWithAncestry`), zero
production callers remain. The 7 S8 test sites (per `emit-container-
recursion.test.ts`) are the only thing keeping it alive.

**Fix**: Delete the function + update the 7 test sites to use
`walkContainersWithAncestry(containers, '', cb)`. The callback can
ignore the second arg for leaf-only use cases.

**Why keep `walkContainersWithAncestry` and not consolidate**: It is
the canonical walker (H3 chose it for parity); the ancestry parameter
is a generalization. Forcing callers that don't need ancestry to
ignore the second arg is cleaner than maintaining two parallel
recursive helpers.

### R-2 — Thread `moduleHeader` from BSWMD

**Current state** (verified by verifier):

- `mcu.ts:279`: `moduleHeader: 'Mcu/Mcu_Cfg.h'` — hardcoded literal
- `mcu.ts:287-288`: `path: 'Mcu/Mcu_Cfg.h'` / `path: 'Mcu/Mcu_Cfg.c'`
  — artifact paths also hardcoded
- `ecuc.ts:383`: `moduleHeader: 'EcuC/EcuC_Cfg.h'` — same pattern
- `ecuc.ts:398`: artifact path hardcoded
- `_shared.ts:144`: `moduleHeader?: string` is the parsed field
- v1.14.1 G1 surfaces `<HEADER>` from BSWMD into `BswmdIndexForModuleHeaderPaths`
- v1.14.2 H2 wired `buildSelfIncludes` to consume `modDef.includes`
  but **NOT** `modDef.moduleHeader` for the template's `{{moduleHeader}}` field

A Mcu BSWMD with `<HEADER>Mcu/McuAlt_Cfg.h</HEADER>` would silently
emit `#include "Mcu/Mcu_Cfg.h"` in the source even though BSWMD
declared `McuAlt_Cfg.h`.

**Fix**:

```ts
// mcu.ts:279
moduleHeader: mDef.moduleHeader ?? `${mDef.shortName}/${mDef.shortName}_Cfg.h`

// mcu.ts:287
path: mDef.moduleHeader ?? `${mDef.shortName}/${mDef.shortName}_Cfg.h`
path: `${mDef.shortName}/${mDef.shortName}_Cfg.c`
```

```ts
// ecuc.ts:383 + :398 — same shape
moduleHeader: eDef.moduleHeader ?? `${eDef.shortName}/${eDef.shortName}_Cfg.h`
```

The `.c` artifact path stays hardcoded — it's always the conventional
`{Module}/{Module}_Cfg.c`, not a BSWMD-controlled path.

**Why a `??` fallback, not a strict require**: Backwards compatibility
with BSWMDs that omit `<HEADER>`. Convention-based default matches the
historical fixture shape (`ecuc-bswmd.arxml`, `mcu-bswmd.arxml`).

### C-1 — `BSW-SEC-004` direct unit test

**Current state** (verified by verifier):

- `diagnostics.ts:31`: `BSW_SEC_MISSING_TARGET_HEADER` declared
- `ecuc.ts:335`, `mcu.ts:239`: pushed only when generator runs
- No test asserts the diagnostic is produced for a BSWMD index entry
  whose target module is loaded but has no `<HEADER>`
- v1.14.2 H2 path is tested indirectly (Refs-1 fixture has Os with
  header), but BSW-SEC-004 push is not directly asserted

**Fix**: New `src/core/generator/__tests__/bsw-sec-004.test.ts`:

```ts
describe('BSW-SEC-004 — EcuCGenerator.emit pushes BSW_SEC_MISSING_TARGET_HEADER', () => {
  it('when referenced module is loaded but has no moduleHeader', () => {
    const def = makeEcucDef({ shortName: 'EcuC' });
    const values = makeEcucValues({
      references: [
        { path: 'EcuC/OsRef', targetModule: 'Os', targetPath: 'Os/OsCoreRef' },
      ],
    });
    const ctx = makeCtx({
      bswmdIndex: new Map([
        ['Os', { shortName: 'Os' }],  // no moduleHeader!
      ]),
    });
    const result = EcuCGenerator.emit(def, values, ctx);
    expect(result.diagnostics).toContainEqual(
      expect.objectContaining({ code: 'BSW-SEC-004' })
    );
  });

  it('does NOT push when referenced module has moduleHeader', () => {
    // ...with moduleHeader set, BSW-SEC-004 is silent
    expect(result.diagnostics).not.toContainEqual(
      expect.objectContaining({ code: 'BSW-SEC-004' })
    );
  });
});
```

Two cases minimum: push + no-push. Mirror for Mcu.
