## Per-Task TDD Pattern

Each task follows the RED → GREEN → IMPROVE pattern from v1.17.0:

1. Write test file FIRST (RED, may be compile-error per v1.6.9+ pattern)
2. Implement minimum to pass (GREEN)
3. Run `pnpm verify` to confirm 8-stage green
4. Dispatch code-reviewer agent → fix CRITICAL/HIGH
5. Commit + push to feature branch
6. Move to next task

## Risk Mitigations

### T2 (SE-1) HIGH RISK

Any preload bridge regression surfaces as renderer crash immediately. Mitigations:

- Feature flag wrapper (`Settings:se1.enabled`, default true in v1.18.0)
- Smoke test in `sandbox-flip.test.ts` (preload audit)
- Staged rollout: ship with feature flag ON, can flip OFF via settings if regression
- Pre-flight: review preload/index.ts line-by-line for any non-typed exposure

### T4 (C13) MEDIUM RISK

Pure refactor with no behavior change. Mitigation:

- Zero consumer change via barrel re-export (§2.1 + §2.2 in spec)
- 5+7 test smoke per sub-component (catches gross regressions)
- Run `pnpm verify` after every sub-component split (incremental validation)

### T7 (PB-4) LOW RISK

New component, doesn't modify existing renderer code. Only modifies `main.tsx` to add wrapper. Mitigation: 3-test suite + visual smoke.

### T8 (C8) MEDIUM RISK

Couples to T1's StepWarning shape. If T1's interface changes after T8 starts, T8 breaks. Mitigation: T1 interface LOCKED after T1 commit.

## Ship Method

1. All T1-T8 commits on `feature/v1-18-0-minor` branch
2. T9 release artifacts commit
3. `git push -u origin feature/v1-18-0-minor`
4. `gh pr create --base main --title "v1.18.0 MINOR — Batch 2 + Batch 3 (8 items)"`
5. PR review (auto-merge if green)
6. `git checkout main && git pull`
7. `git tag v1.18.0` → `git push origin v1.18.0`
8. `gh release create v1.18.0 --target main --notes-file docs/release-notes/v1.18.0/README.md --title "v1.18.0 — Batch 2 + Batch 3"`
9. If network flaps: use `gh api` 5-step chain per v1.17.0 pattern
