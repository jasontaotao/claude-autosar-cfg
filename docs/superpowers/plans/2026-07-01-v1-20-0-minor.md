**Goal:** Close 2 remaining deferred items (T4 C2.4 GUI applyMutation parity + B-3 second half template reshape) in a single MINOR release. **2 items + 1 housekeeping + 1 release = 4 commits.**

**Baseline:** v1.19.1 PATCH `01fb0fb` on `main` (T1-T3 of the original v1.20.0 scope already shipped in v1.19.1 PATCH commits `42b1942` + `202f2d5` + `fecbadb`).

**Architecture:**

- **T1 (C2.4)** behavioral rewrite: `useScriptStore.applyMutation` routes through `applyPatchSteps` (same engine as CLI). Add `warnings` field to `ScriptRunResult`. Delete ~70 lines of duplicated GUI helpers.

**Tech Stack:** TypeScript 5.x (ESM, `"type": "module"`), Handlebars 4.x, Electron 30, Node ≥22.13, Vitest 1.6, Vite 5, ESLint 8, Prettier. **No new top-level deps.**

**Ship pattern:** direct on `main` (per v1.18.0+ cycle), 4 commits (T1-T4). Pre-ship joint code-review (per `claude-autosarcfg-v1-13-x-joint-review` methodology). Ship via `gh api` per `git-push-network-workaround` pattern.

- happy path: 1 set-param mutation → `applyPatchSteps` called → `warnings=[]` → `useArxmlStore.setDoc` called → `projectSave` IPC called once → `mutations: []` cleared → `dirty: false`
- warnings threaded: a step that emits a `StepWarning` (e.g. `variant-downgrade` via `applyPatchSteps` direct call) → `runResult.warnings` populated with `{ stepIndex, kind, message }`

- IPC save failure: `projectSave` returns `write-failed` → `runResult.errorMessage` set + `dirty: true` (in-memory doc still updated)
- no active doc → stub contract preserved (mutations=[], dirty=false)

Update the existing H2 test (`useScriptStore.test.ts:701` "remove-child on a path with active references is surfaced (H2)") to assert the new error message format: `applyPatchSteps` returns an error of kind `name-conflict` or `path-not-found` (depending on `coreRemoveWithCascade` behavior with refs), and `runResult.errorMessage` reflects it. `pendingDelete` is `null` (no dialog ever opened).
